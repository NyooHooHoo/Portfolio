Object Oriented Programming (OOP)

Procedural Programming is a basic paradigm of programming taught to beginners, where one main program simply runs start to finish. In Java, a more complex but very useful form of programming used is Object Oriented Programming (OOP), where objects implemented to store and manipulate data.

An Object is any set of data representing real-world objects or abstract concepts, definted by it's identity (stored memory), state (variables), and behaviour (methods). All objects are instances of a Class, blueprint definitions of objects' structures written by a programmer. Objects are instantiated from classes using a Constructor, a method with the same name as the class itself that creates a new object in memory when used with the new keyword. All constructors may accept certain parameters that allow users to specify the data stored in the new object. Overloading is the use of several constructors for same class with different combinations of such parameters.

public Elemental(String name, int age, double maxMana, double maxHealth, double mana, double health, String elementalForm, boolean isFriendly) {
    this.name = name;
    this.age = age;
    this.maxMana = maxMana;
    this.maxHealth = maxHealth;
    this.mana = mana;
    this.health = health;
    this.elementalForm = elementalForm;
    this.isFriendly = isFriendly;
}
    
public Elemental(String form) {
    name = "Steve";
    age = 0;
    maxMana = 100;
    maxHealth = 100;
    mana = 100;
    health = 100;
    elementalForm = form;
    isFriendly = true;
}
     
public Elemental() {
    name = "Steve";
    age = 0;
    maxMana = 100;
    maxHealth = 100;
    mana = 100;
    health = 100;
    elementalForm = "Qarri";
    isFriendly = false;
}

Constructors from the Elemental class from the Creation Story assignment are overloaded with a custom constructor, with parameters for each field, a default constructor, and a special constructor with only one parameter to be used within the program. Each of these can be used to instantiate a unique Elemental object (e.g. Elemental e = new Elemental();)

Static members are variables and methods of a class independent from its objects, definted using the static keyword. Static variables only exist once across all objects of class, and when it's value changes all objects are affected. Static methods can be accessed directly from the class without an object to call it from. All non-static members of a class are accessed using dot notation (objectName.memberName).

private static String[] gasses = new String[0];

/* Constructor and other methods not shown */

public static String getGasses(){
    String message = "Current Gasses in the atmosphere:" + "\n";
    for (int i = 0; i < gasses.length; i++)
        message = message.concat(gasses[i] + "\n");
    return message;
}

Snippet from Hyrmus class in Creation Story assignment. The gasses variable is set to static to reflect there only being one composition of gasses in the atmosphere that should change across all objects of the class. As such, the getGasses() accessor is also a static method to be called by the class in general instead of any one object (i.e. Hyrmus.getGasses()).

Encapsulation is the process of controlling access to class members to endure data security. Variables and methods with the private keyword can only accessed within their own class, while public ones can also be accessed outside the class. Without direct outside access, get (accessors) and set (mutator) methods are implemented private fields when needed.

Snippet from Airplane class assignment. The instance variable passengers, storing the number of passengers on a plane, is set to private. It can only be accessed by the public accessor getPassengers and the mutators loadPass() and unloadPass(), which limit how the value can be changed from outside the class.

Unified Modeling Language (UML) is a standard representation of a class that can be applied to any object-oriented programming languages, not just Java. A UML table shows class name and all its variables and methods, along with parameters and return values. Plus (+) and minus (-) signs represent public and private members, respectively.

UML of a Warrior class from my first open book test. All variables are private with a leading minus sign and all methods are public with a leading plus. The data type of each variable is labeled accordingly, such as String for name. The parameter names and types, as well as the return types for each method are also labeled, including those of the class constructor.
