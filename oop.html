Object Oriented Programming (OOP)

Procedural Programming: most basic, one main program runs start to finish Object Oriented Programming (OOP): objects implemented to store and manipulate data Object: identity (stored memory), state (variables), and behaviour (methods) Class: blueprint definition of object where they are instantiated from Constructor: method used to create new object with new keyword, same name as class Overloading: several constructors for same class with different parameters

public Elemental(String name, int age, double maxMana, double maxHealth, double mana, 
double health, String elementalForm, boolean isFriendly) {
    this.name = name;
    this.age = age;
    this.maxMana = maxMana;
    this.maxHealth = maxHealth;
    this.mana = mana;
    this.health = health;
    this.elementalForm = elementalForm;
    this.isFriendly = isFriendly;
}
    
public Elemental(String form) {
    name = "Steve";
    age = 0;
    maxMana = 100;
    maxHealth = 100;
    mana = 100;
    health = 100;
    elementalForm = form;
    isFriendly = true;
}
     
public Elemental() {
    name = "Steve";
    age = 0;
    maxMana = 100;
    maxHealth = 100;
    mana = 100;
    health = 100;
    elementalForm = "Qarri";
    isFriendly = false;
}

Constructors from the Hyrmus class from the Creation Story assignment are overloaded with a custom constructor, with parameters for each field, a default constructor, and a special constructor with only one parameter to be used within the program.

Static: keyword for parts of class independent from objects, procedural programming static variables only exist once across all objects of class static methods can be accessed without an object All non-static members accessed using dot notation

Hyrmus.java Copy 
private static String[] gasses = new String[0];

/* Constructor and other methods not shown */

public static String getGasses(){
    String message = "Current Gasses in the atmosphere:" + "\n";
    for (int i = 0; i < gasses.length; i++)
        message = message.concat(gasses[i] + "\n");
    return message;
}

Snippet from Hyrmus class in Creation Story assignment. The gasses variable is set to static to reflect there only being one composition of gasses in the atmosphere that should change across all objects of the class. As such, the getGasses accessor is also a static method to be called by the class in general instead of any one object.

Encapsulation: security by controlling outside access to class members private only accessed within class, public can also be accessed outside class Prevents direct access of data from outside, can only access through get (accessors) and set (mutator) methods

Airplane.java Copy 
private int passengers;

/* Constructor and other methods not shown */

public void loadPass(int newPass) {
    passengers += newPass;
}

public void unloadPass() {
    passengers = 0;
}

public int getPassengers() {
    return passengers;
}

Snippet from Airplane class assignment. The instance variable passengers, storing the number of passengers on a plane, is set to private. It can only be accessed by the public accessor getPassengers and the mutators loadPass() and unloadPass(), which limit how the value can be changed from outside the class.

Unified Modeling Language (UML): a standard representation of a class Shows class name and all variables and methods (with parameters and return) Plus (+) and minus (-) signs represent public and private

UML of a Warrior class from my first open book test. All variables are private with a leading minus sign and all methods are public with a leading plus. The data type of each variable is labeled accordingly, such as String for name. The parameter names and types, as well as the return types for each method are also labeled, including those of the class constructor.
