ArrayList

One limitation of default arrays in Java is that the length at which they are initialized can never be changed, making it difficult to add or remove elements from the array. However, if we import the java.util package into our code, we can use the ArrayList class instead, which allows for resizable lists of elements. After being imported, an ArrayList can be initialized using the following syntax:

ArrayList<E> name = new ArrayList<E> (capacity)

E is the class that all elements of the list will be objects of, and capacity is the maximum number of elements the list can hold, which is 10 by default. This is different from the number of elements actually stored in the list, which can be accessed by name.size(). The capacity of an ArrayList automatically doubles every time the number of elements reaches it. Some other basic methods for the ArrayList class are add(), which adds an element to the end of the list by default or a specified index, and get() which accesses the element at a specified index.

    player3 = new ArrayList<>();

Instance variables and class constructor for the Bridge class from ArrayList assignment. Five ArrayLists variables storing String objects are initialized with default capacity. The add() method is used to populate deck from deckString.

ArrayLists only hold object types such as Integer or Double , not primitive data types such as int or double, but Java automatically converts primitive data types to objects when passed into an ArrayList in a process called autoboxing. For example list.add(12) would run without error on a variable declared as ArrayList<Integer> list; even though a primitive int is being passed as an Integer object.

The For-each loop is a new enhanced for loop introduced in Java 5 that allows for more convenient and readable traversal of arrays and ArrayLists. This type of loop creates a variable directly referencing the values of elements themselves, instead of looping through index values and accessing elements in the loop body. This is done using the syntax for (type element : arr).

Common Mistakes

  • Forgetting to import java.util.*; at the top of the program before using ArrayList.
  • Initializing with a primitive data type such as ArrayList<int> instead of a class such as ArrayList<Integer>.
  • Using length or length() to access the number of elements in an ArrayList instead of size().
  • Using square brackets list[0] to access an element in an ArrayList instead of the method list.get(0).
  • Treating the looping variable in a for-each loop as an index instead of an element value. For example, for (int i : list) System.out.println(list.get(i)); should simply print i instead.
