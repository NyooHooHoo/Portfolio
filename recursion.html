Recursion

Recursion is a technique in Java where a method calls itself within its own body, turning them into recursive methods. By doing this, a recursive method is able to solve large and complex tasks by continuously divinding them into smaller tasks. To do this, a recursive method must be able to handle two cases: the base case and the recursive case.

The base case, also called the halting condition, represents the smallest possible division of a task and when the method no longer calls itself. This is crucial to preventing the method from preventing the the method from calling infinitely. On the other hand, the recursive case handles all other cases where the method does call itself again, continuing the recursion.

public static int exponent(int b, int e) {
    int result = 1;
    for (int i = 0; i < e; i++)
        result *= b;
    return result;
}

public static int exponentR(int b, int e) {
    if (e == 1)
        return b;
    return b * exponentR(b, e - 1);
}

Two static methods from Recursion Assignment Set 1 that show both an iterative (regular looping) and recursive solution to calculating an exponential given a base and exponent. The iterative method exponent() uses a for loop to multiply the integer b by itself until the counter reaches the integer e. The recursive method exponentR() has a base case of e == 1, because any number to the power of 1 is always itself, and a recursive case of multiplying b by itself to the power of one less than the passed exponent, which would eventually reach the base case and return the correct value.

public static int minCost(int[][] grid, int r, int c) {
    if (r == 0 && c == grid[0].length - 1)
        return grid[r][c];
    else if (r == 0)
        return grid[r][c] + minCost(grid, r, Math.min(c + 1, grid[c].length-1));
    else if (c == grid[0].length - 1)
        return grid[r][c] + minCost(grid, Math.max(0, r - 1), c);
        
    int north = minCost(grid, Math.max(0, r - 1), c);
    int east = minCost(grid, r, Math.min(c + 1, grid[c].length - 1));
    return grid[r][c] + Math.min(north, east);
}

A more complex application of recursion from Recursion Assignment Set 2, where the method minCost() is used to find the minimum cost to travel from one corner to another in a grid of integers. The method has 3 base cases for when the north edge, east edge, and north-east corner have been reached, in which case there is only one possible value to return. The recursive case of the method calls minCost() starting from the position 1 north and again 1 east of the current position. It then adds the minimum of these two values to the final cost, allowing the method to always find the lowest cost after each layer of recursion.

Common Mistakes

  • Forgetting to define a base case in a recursive method, causing it to call itself infinitely and triggering a stack overflow, an error where the computer runs out of memory to store all the method calls.
  • Not updating the values passed into the method each recursive call to reach the base case, causing a stack overflow even if a base case is defined.
