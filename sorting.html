Searching & Sorting

Two important concepts that are often used in programming are searching and sorting. Firstly, searching is the process of looking through the data stored in an Array, ArrayList, or other data structure to find a specific value. There are two common algorithms, or sets of instructions, used to perform a search: linear search and binary search.

A linear search is the simplest searching algorithm, checking every element in the array or list until the desired value is found. As such, the number of checks that must be made is directly proportional to the size of the list. This level of efficiency is expressed by programmers in Big-O notation as O(n).

The changeMark() method from the MaCSBook assignment uses a very simple for loop to to find the index of a specific student number in the list numbers by linear search.

Unlike a linear search, a binary search can only be used in an already sorted array, but is much faster with O(log n) efficiency. Instead of checking every individual value, a binary search starts in the middle of the sorted array and checks if it is higher or lower than the value being searched for. Depending on this, the next value the algorithm checks is the middle value of the left or right half of the array. This process continues until the exact value is found, requiring fewer searches than linear search.

The above animation from Penjee shows the difference in process and efficiency between a linear and binary search on the same set of data. The binary search only required 3 steps while the linear takes 11.

Sorting is the process of arranging a set of data in ascending or descending order based on the values of the elements. This allows methods such as binary search to be performed. Like with searching, there exist many different sorting algorithms, each with their own specific traits, and the efficiency of sorting algorithms can also be measured using Big-O notation.

Insertion sort is one of the simpler sorting algorithms that "inserts" elements one at a time into a sorted order built from left to right within the original array, similar to how one would sort a hand of playing cards. This is done through iterating through each value and continuously swapping it with the value before it until it is in sorted position. This two-layered iteration makes insertion sort not very fast for large sets of data, with an O(n2) efficiency.
